新一代互联网-绪论

发表于 2018-09-14 | 更新于 2018-09-15

1. NGI的由来

NGI即Next Generation Internet，下一代互联网

1.1 Internet的出现和发展

  • 1969年出现，美国军方建设的ARPANET
  • 1982年，TCP/IP被美国军方接受
  • 1983年，ARPANET被划分为两部分：MILNET（军用）和ARPANET（民用）
  • 1988年，美国国家科学基金会（NSF）建立的NSFNET取代ARPANET

  • 1989年，ARPANET被撤销，Internet正式民用

    如今规模

  • 截至2018年6月30日，中国网民规模达8.02亿，互联网普及率为57.7%。

1.2 NGI的提出

需求：

  • 用户数量不断增长，IPv4的不足逐渐显现

    IPv4的不足现阶段主要通过使用双协议栈、隧道技术、网络协议转换、地址转换（如NAT）等方式解决

  • 用户在地址空间、服务质量、移动和安全等方面提出更高要求

提出：

  • 克林顿政府1996年提出“下一代互联网计划”是NGI的开始
  • NGI到现在为止没有形成统一和确切的定义

对NGI的需求和基本特征的共识：

  • 更大、更快、更及时、更方便、更安全、更易管理和更有效益
    • 更大：过渡到地址空间巨大的IPv6
    • 更快：传输速率应提高1000倍以上
    • 更及时：QoS(Quality of Service)， 组播
    • 更方便：接入方式更方便快捷（如移动接入）
    • 更安全：网络安全可信
    • 更易管理：可管可控，易于重组和重构
    • 更有效益：
      • 要有好的经济模型，形成好的盈利模式
      • 改变目前基础网络运营商投资巨大盈利甚微甚至亏损而增值业务开发商投资商盈利丰的局面
  • 扩展性、高性能、实时性、移动性、安全性、管理和经济性
    • 扩展性：终端从计算机扩展到可连接的所有电子设备
    • 高性能：更高的传输速率支持更高性能的互联网应用
    • 实时性：QoS控制能力强、有保障，非当前“尽力而为”策略
    • 移动性：无处不在，无时不在
    • 安全性、管理、经济性基本对应上面

1.3 国外NGI的相关研究

  1. 1992年美国提出“信息高速公路”

  2. 2002年¨美国Internet2和欧洲GéANT实现5Gbps高速互联

  3. 2005年美国启动两项NGI研究计划

    • FIND(Future Internet Design )：满足未来15年社会需求的新型网络

    • GENI (Global Environment for Network Innovations)：崭新的、高安全的、能够连接一切设备的互联网

  4. 2007年欧盟启动FIRE项目

    未来互联网研究和实验（Future Internet Research and Experimentation）

  5. 2010年美国启动FIA(FUTURE INTERNET ARCHITECTURE )研究计划

  6. 2014年美国启动FIA-NP(Next Phase)

1.4 国内NGI的研究

  1. 20世纪90年代后期开始，中国启动了一系列与GNI研究有关的计划

    • 教育部建设了CERNET-IPv6试验网
    • 国家自然科学基金委员会资助建设了中国高速互联研究试验网络（NSFCNET）
    • 863计划支持了高性能宽带信息网 3TNET等重大专项
    • 中科院开展“IPv6关键技术以及城域示范网 ”
    • 原国家计委组织启动“下一代互联网中日IPv6合作项目”

  2. 2003年CNGI正式启动

    • 中国NGI示范项目（China Next Generation Internet）
    • 6个主干网：
      • CNGI-CERNET2是CNGI中唯一的学术网，也是其中规模最大的主干网，2004年开通并提供服务

2. NGI与NGN

NGN，Next Generation Network，下一代网络

2.1 NGN的提出

  • 网络融合的需求促进了下一代网络的诞生

  • 广义的NGN包括：

    • 基础传送层：智能光网络
    • 承载层：以MPLS和IPv6为核心的、具有QoS保障的、安全的分组网络
    • 接入层：多元化，综合化，宽带有线与无线接入
    • 网络控制层：软交换
    • 移动通信角度：3G和后3G、4G、5G发展范畴
    • 业务角度：开放的智能化综合业务平台

  • 国际电信联盟给出的基本定义：

    基于分组技术的网络，业务相关功能与底层传送相关技术相互独立，能够提供包括电信业务在内的多种业务，能够利用多种带宽和具有QoS支持能力的传送技术，能够使用户自由接入不同的业务提供商，能够支持广泛移动性（如终端／个人／会话／服务／模式／组网设备／网络），向用户提供一致的、无处不在的业务

  • NGN的出现表面电信网络从以电路交换为主的传统网络逐渐迈向分组交换网络

  • 以软交换为核心，采用开发、标准的体系结构，向分组化、宽带化、融合化、智能化发展，同时保持与现有网络互联互通

2.2 国内外研究

  1. 国外

    主要集中在ITU等组织中，并在欧美和日本等国得到了广泛实施。2008年日本的NTT的NGN网络正式投入运营

  2. 国内

    • 20世纪90年代，中国启动了对NGN的研究
    • 中国电信于2000年开始了我国最早的NGN试验工程，2003年完成了全部现场技术试验和NGN软交换业务试验
    • 中国网通于2005年10月启动了面向NGN的全网升级计划
    • 中国移动主要针对NGN与3G业务的融合进行研发
    • 华为、中兴、大唐等提出了相应的NGN解决方案

2.3 NGI与NGN的异同

  • NGN产生于电信界，NGI产生于计算机界
  • 出发点虽不同，但目标相同
    • NGI的出发点是提供数据业务的无QoS保证的Internet
    • NGN的出发点是提供语音业务的有QoS保证的PSTN
    • 但其共同的目标是全业务网络
  • 网络管理与控制方面
    • NGN强调网络集中控制与管理
    • NGI网络管理和控制更倾向于分布式、非集中化
  • 智能网络和智能终端
    • NGN认为网络是智能的
    • NGI中智能分散在终端
  • 编号与寻址
    • NGN更关注网络寻址和编号的统一问题，特别是电话网和Internet编号和寻址的统一
    • NGI关注纯IP域的寻址
  • 计费
    • NGN更关注业务、计费与商业模式
    • NGI更侧重学术研究性质
  • QoS需求
    • NGN以电信业务为主，强调网络安全与QoS保证
    • NGI强调支持QoS，但不提供保证
  • 不排斥多个网络联合实现的方式：数据业务可通过NGI承载，电信业务可通过NGN承载

3. NGI的体系结构

对网络的研究可分为体系结构、协议标准、机制算法三个部分

3.1 对体系结构有两种基本的研究思路

  • 基于现有的Internet体系结构进行改进
  • 重新设计新的Internet体系结构

3.2 多维可扩展的NGI体系结构

  • 多维可扩展：规模，性能，服务，安全，功能
  • 基本要素：IPv6协议，真实地址访问，可扩展网络节点，端到端无连接服务质量，IVI过渡策略
